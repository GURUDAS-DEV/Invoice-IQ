LANDING SELLER PAGE PROMPT : 

Design a modern SaaS-style Seller Management Page for a web application called Invoice IQ, which helps small shop owners track purchases from suppliers and monitor price changes.

The design must follow these UI principles:

Clean modern SaaS platform design

Support dark mode and light mode

Use Lucide icons

Minimal, professional layout (not flashy or cluttered)

Smooth spacing and clear hierarchy

Designed for fast usability for busy shopkeepers

Assume the application already has a left sidebar navigation with the following links:

Dashboard

Sellers

Deliveries

Analytics

Reports

The Sellers page should be the main content area.

Page Layout

The Sellers page should contain two main sections:

Page Header

Seller Management Area

1. Page Header

At the top of the page display:

Title:
"Sellers"

Subtitle:
"Manage suppliers and track purchase history."

On the right side of the header, add a primary action button:

Button:
"Add Seller"

Use a Lucide Plus icon.

Button style:

Primary SaaS button

Rounded corners

Slight shadow

Clear hover state

Clicking this button should open a modal or slide-down form to add a seller.

2. Add Seller Form (Modal or Expandable Section)

When the user clicks Add Seller, show a clean form with these fields:

Seller Name (required)
Location (optional)
Contact Number (optional)

Form design requirements:

Clean input fields

Proper spacing

Clear labels

Modern SaaS styling

At the bottom of the form include two buttons:

Cancel
Save Seller

Save Seller should be the primary button.

3. Seller List Section

Below the header, display a seller list table or card grid showing all existing sellers.

Each seller card or row should display:

Seller Name
Location
Last Delivery Date
Total Spend This Month

Include small Lucide icons next to important data fields.

Example:

Store icon → Seller Name
MapPin icon → Location
Calendar icon → Last Delivery
Wallet icon → Monthly Spend

4. Seller Card / Row Interaction

Each seller entry should be clickable.

When clicked, it will navigate to the Seller Detail Page (this page will be implemented later).

For now, just design it as a clickable element.

Hover state should slightly elevate the card or highlight the row.

5. Empty State (Important)

If there are no sellers yet, show a friendly empty state.

Display:

Icon: Lucide Store icon

Message:

"No sellers added yet."

Subtext:

"Add your first supplier to start tracking purchases."

Include a primary button:

"Add Seller"

6. Search and Filtering

Above the seller list include a search bar.

Placeholder text:

"Search sellers..."

Optionally include a small filter dropdown for:

All Sellers
Recently Active
Inactive

7. Visual Design Guidelines

The page must follow a modern SaaS design similar to platforms like:

Stripe

Linear

Notion

Use:

soft shadows

rounded cards

clean typography

balanced spacing

subtle hover animations

Avoid clutter and flashy colors.

Product Detail Page UI Prompt :

Design a modern SaaS-style Product Detail Page for a web application called Invoice IQ, which helps small shop owners track product purchases from suppliers and analyze price changes over time.

The interface must follow these design principles:

Clean modern SaaS layout

Support dark mode and light mode

Use Lucide icons

Minimal, professional styling

Easy to scan and read quickly

Avoid flashy or cluttered design

Assume the application already has a left sidebar navigation with:

Dashboard

Sellers

Deliveries

Analytics

Reports

This page opens when a user clicks a product from a seller's product list.

Page Layout

The Product Detail Page should contain four main sections:

Product Header

Supplier Information

Purchase History Controls

Purchase History Table

Price Trend Analytics

1. Product Header

At the top left, display the product information.

Show the Product Name prominently in a large bold heading.

Example:

Basmati Rice

Above or beside the title include a Back button that navigates back to the seller detail page.

Use a Lucide ArrowLeft icon.

2. Supplier Information

Below the product title show the supplier information.

Label:

Supplier

Display the seller name from whom the product is purchased.

Example layout:

Product Name
Supplier: Guptaji Traders

Use Lucide Store icon beside the supplier name.

This section should be compact but clearly visible.

3. Purchase History Controls

Below the header create a Purchase History Filter Section.

Title:

Purchase History

This section allows the user to filter historical purchases.

Provide two filtering methods, but only one can be active at a time.

Option 1 — Time Range Filter

Provide quick buttons for time-based filtering.

Buttons:

Today
Last Week
Last Month
Last 6 Months
Last Year
Last 2 Years
Last 5 Years

When one is selected, others should automatically deselect.

Use a segmented control or filter chips style UI.

Option 2 — Specific Month Selection

Provide a month selector dropdown or calendar picker.

Example:

Select Month: March 2026

When this option is used, the time range filters should automatically disable.

This allows users to view all purchases made during a specific month.

4. Purchase History Table

Below the filter section display the Purchase History Table.

Each row represents one delivery or purchase event for that product.

Table columns should include:

Date
Quantity
Unit
Price per Unit
Total Price
Source

Example row:

15 March 2026
50
kg
₹120
₹6000
Invoice Upload

Use small Lucide icons where appropriate:

Calendar icon → Date
Package icon → Quantity
Tag icon → Price
FileText icon → Source

The table should support scrolling if there are many entries.

5. Price Trend Analytics

Below the purchase history table create an Analytics Section.

Title:

Price Trend

Display a line chart showing how the price of this product has changed over time.

The chart should visualize:

X-axis → Time (months or dates)
Y-axis → Price per unit

Use Chart.js for the graph implementation.

The graph should show a smooth price trend from the first recorded purchase to the most recent purchase.

Optional enhancements:

Hover tooltips showing price and date

Highlight highest and lowest prices

Example tooltip:

March 2026 — ₹120/kg

Visual Design Guidelines

Follow the design style of modern SaaS products such as:

Stripe

Linear

Notion

Use:

soft shadows

rounded cards

subtle hover states

clear typography hierarchy

balanced whitespace

Avoid overly complex UI elements.

Analytics Page UI Prompt

Design a modern SaaS-style Analytics Page for a web application called Invoice IQ, a platform that helps small shop owners track supplier purchases, monitor price changes, and analyze buying patterns.

The UI must follow these design principles:

Modern SaaS dashboard style

Support dark mode and light mode

Use Lucide icons

Clean layout with strong visual hierarchy

Minimal design (not flashy or cluttered)

Focus on data insights and charts

Assume the application already has a left sidebar navigation with the following links:

Dashboard

Sellers

Deliveries

Analytics

Reports

This page should represent the Analytics section where users can explore deeper insights about their purchases and suppliers.

Page Layout Structure

The Analytics Page should contain five main sections:

Page Header and Filters

Key Insight Summary Cards

Spending Over Time Chart

Supplier Spending Comparison

Top Purchased Products

Price Change Insights

1. Page Header and Global Filters

At the top display:

Title:

Analytics

Subtitle:

"Insights about your purchases, suppliers, and pricing trends."

On the right side include a time range filter dropdown.

Options:

Last 7 Days
Last Month
Last 6 Months
Last Year
Custom Date Range

Changing the filter should update all charts and insights on the page.

2. Key Insight Summary Cards

Below the header display four key analytics cards in a row.

Each card should have:

icon

label

large numeric value

Use Lucide icons.

Cards include:

Total Spending

Icon: Wallet

Example value:

₹1,45,000

Shows total amount spent on purchases within selected time range.

Active Suppliers

Icon: Store

Example value:

8 Suppliers

Number of suppliers used during the selected time range.

Total Deliveries

Icon: Truck

Example value:

42 Deliveries

Total number of delivery records added.

Most Purchased Product

Icon: Package

Example value:

Basmati Rice

The product with highest quantity purchased.

Cards should have:

rounded corners

subtle shadows

soft hover effect

clean spacing

3. Spending Over Time Chart

Below the summary cards create the primary analytics chart.

Title:

Spending Over Time

Use a line chart to show how total spending changes across time.

X-axis:

Months or dates

Y-axis:

Total spending amount

Example:

January – ₹20,000
February – ₹18,500
March – ₹27,000

Implement the chart using Chart.js.

Add tooltips on hover to show exact values.

4. Supplier Spending Comparison

Below the first chart create a bar chart comparing spending across suppliers.

Title:

Supplier Spending Comparison

Each bar represents a supplier.

Example:

Sharma Traders – ₹40,000
Gupta Suppliers – ₹25,000
Verma Wholesalers – ₹18,000

This helps users understand which supplier receives the most spending.

Use distinct colors but keep them subtle to match SaaS design.

5. Top Purchased Products

Below the supplier chart create a horizontal bar chart.

Title:

Top Purchased Products

Shows products with highest purchased quantities.

Example:

Rice – 500 kg
Sugar – 320 kg
Dal – 210 kg
Oil – 180 L

This helps shopkeepers understand which products they buy most frequently.

Use Chart.js horizontal bar chart.

6. Price Change Insights

At the bottom of the page add a section called:

Price Change Insights

Subtitle:

"Recent price changes detected from suppliers."

Display a list of detected price changes.

Example insights:

⬆ Mustard Oil price increased by ₹10/L since last purchase
⬆ Basmati Rice price increased by ₹15/kg compared to last month
⬇ Sugar price decreased by ₹2/kg from Gupta Suppliers

Use color coding:

Red tone for price increases
Green tone for price decreases

Use Lucide icons:

TrendingUp icon → price increase
TrendingDown icon → price decrease

Each insight should appear as a small alert-style card or list row.

Visual Design Guidelines

Follow the visual style of modern SaaS analytics platforms such as:

Stripe

Linear

Notion

Design requirements:

soft shadows

rounded cards

subtle hover interactions

balanced whitespace

clean typography hierarchy

Avoid overly complex visuals.
